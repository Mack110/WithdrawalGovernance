## Construction Methodology

### Prompt Generation Method

**Primary Generation:**
- Prompts were generated using a large language model (GPT-4) under controlled instructions
- Researchers provided detailed rule definitions and category specifications to the LLM
- Each generated prompt was manually reviewed by researchers to verify:
  - Correct category assignment (ALLOW or specific violation rule)
  - Clarity and coherence of text
  - Alignment with WithdrawalRuleSet-v0.1.md definitions
  - Absence of obvious artifacts or errors

**Paraphrasing & Variation:**
- Paraphrases were generated by the large language model to test rule brittleness
- For each primary prompt, 1–3 semantic variations were created using LLM paraphrasing
- Variations maintain the same semantic violation category (or benign status)
- Purpose: Assess whether rules detect violations despite linguistic variation

**Adversarial Design:**
- Prompts near rule boundaries were intentionally included (e.g., "defer to my expertise" vs. "follow my judgment unconditionally")
- Researchers selected borderline cases to test rule precision and clarity
- No adversarial red-teaming against known rule implementations was conducted during generation

- ALLOW cases: ~25–27% of each split (realistic expectation in deployment)

## Permitted Uses and Non-Uses

### Permitted
✅ Evaluate rule detection accuracy on synthetic prompts  
✅ Analyze failure modes and error distributions  
✅ Publish findings on rule brittleness or effectiveness  
✅ Use as benchmark for other withdrawal-rule implementations

### Not Permitted
❌ Deploy rules directly to production without additional safety evaluation  
❌ Claim generalizability to real-world user interactions  
❌ Use as training data for new models without explicit consent and separate versioning  
❌ Represent synthetically-derived performance metrics as guarantees on real data

## Data Quality and Bias Notes

### Known Limitations
- **Synthetic origin:** Prompts reflect author writing style and assumptions
- **Language:** English-only; may not generalize to other languages
- **Author bias:** Categories reflect researcher judgment on boundary cases
- **Linguistic diversity:** Limited variation in sentence structure, register, dialect
- **Cultural context:** Autonomy framing is Western/individualistic

### Recommendations for Extended Datasets
1. Diversify authorship across geographies and disciplines
2. Include multilingual prompts and translations
3. Gather real-world examples (with appropriate privacy protections)
4. Validate category labels via crowdsourcing with clear rubric
5. Conduct adversarial robustness testing against iteratively-learned attacks
